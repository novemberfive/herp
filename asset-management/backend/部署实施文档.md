# 资产管理系统 - 后端部署实施文档

## 一、系统要求

### 1.1 硬件要求（轻量级配置）
- CPU: 2 核及以上
- 内存：2GB 及以上
- 磁盘：10GB 及以上
- 网络：10Mbps 及以上

### 1.2 软件要求
- JDK 17+
- MySQL 8.0+
- Maven 3.8+
- Linux (CentOS 7+/Ubuntu 18.04+) 或 Windows Server 2016+

---

## 二、环境准备

### 2.1 安装 JDK 17

#### Linux (CentOS/Ubuntu)
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-17-jdk -y

# CentOS/RHEL
sudo yum install java-17-openjdk-devel -y

# 验证安装
java -version
```

#### Windows
1. 下载 JDK 17: https://adoptium.net/
2. 运行安装程序，按向导完成安装
3. 配置环境变量 JAVA_HOME
4. 验证：`java -version`

### 2.2 安装 MySQL 8.0

#### Linux (Ubuntu)
```bash
sudo apt update
sudo apt install mysql-server -y
sudo systemctl start mysql
sudo systemctl enable mysql
sudo mysql_secure_installation
```

#### Linux (CentOS)
```bash
# 添加 MySQL YUM 仓库
sudo yum install https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm -y
sudo yum install mysql-server -y
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码并修改
sudo grep 'temporary password' /var/log/mysqld.log
sudo mysql_secure_installation
```

#### Windows
1. 下载 MySQL Installer: https://dev.mysql.com/downloads/installer/
2. 运行安装程序，选择"Developer Default"或"Server only"
3. 设置 root 密码（建议：root123）
4. 验证：打开命令提示符，执行 `mysql -u root -p`

### 2.3 安装 Maven

#### Linux
```bash
# Ubuntu/Debian
sudo apt install maven -y

# CentOS
sudo yum install maven -y

# 验证
mvn -version
```

#### Windows
1. 下载 Maven: https://maven.apache.org/download.cgi
2. 解压到 C:\Program Files\Apache\maven
3. 配置环境变量 MAVEN_HOME
4. 验证：`mvn -version`

## 三、数据库初始化

### 3.1 创建数据库并导入数据

```bash
# 登录 MySQL
mysql -u root -p

# 执行初始化脚本
mysql -u root -p < /path/to/backend/src/main/resources/schema.sql
```

或者在 MySQL 客户端中手动执行：
```sql
source /path/to/backend/src/main/resources/schema.sql;
```

### 3.2 验证数据库
```sql
USE asset_db;
SHOW TABLES;
SELECT username, role FROM sys_user;
```

预期看到以下表：
- sys_user
- asset_category
- asset_location
- asset_card
- purchase_apply
- inventory_task

## 四、应用部署

### 方式一：JAR 包直接运行（推荐）

#### 4.1 编译打包
```bash
cd /path/to/backend

# 清理并打包
mvn clean package -DskipTests

# 生成的 JAR 包位置：target/asset-management-1.0.0.jar
```

#### 4.2 配置文件修改
编辑 `src/main/resources/application.yml`，修改数据库连接信息：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/asset_db?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的 MySQL 密码
```

#### 4.3 运行应用
```bash
# 直接运行
java -jar target/asset-management-1.0.0.jar

# 后台运行（Linux）
nohup java -jar target/asset-management-1.0.0.jar > app.log 2>&1 &

# 查看日志
tail -f app.log
```

#### 4.4 验证启动
看到以下日志表示启动成功：
```
========================================
   资产管理系统启动成功！
   访问地址：http://localhost:8080/api
========================================
```

### 方式二：Docker 部署

#### 4.5 创建 Dockerfile
在项目根目录创建 `Dockerfile`：
```dockerfile
FROM openjdk:17-slim

WORKDIR /app

COPY target/asset-management-1.0.0.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 4.6 构建并运行 Docker 镜像
```bash
# 构建镜像
docker build -t asset-management:1.0.0 .

# 运行容器
docker run -d \
  --name asset-backend \
  -p 8080:8080 \
  -e SPRING_DATASOURCE_URL=jdbc:mysql://host.docker.internal:3306/asset_db?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai \
  -e SPRING_DATASOURCE_USERNAME=root \
  -e SPRING_DATASOURCE_PASSWORD=root123 \
  asset-management:1.0.0

# 查看日志
docker logs -f asset-backend
```

## 五、服务管理（Linux）

### 5.1 创建 systemd 服务文件
```bash
sudo vim /etc/systemd/system/asset-management.service
```

内容如下：
```ini
[Unit]
Description=Asset Management System
After=syslog.target network.target

[Service]
User=root
WorkingDirectory=/path/to/backend
ExecStart=/usr/bin/java -jar /path/to/backend/target/asset-management-1.0.0.jar
SuccessExitStatus=143
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 5.2 管理服务
```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start asset-management

# 设置开机自启
sudo systemctl enable asset-management

# 查看状态
sudo systemctl status asset-management

# 停止服务
sudo systemctl stop asset-management

# 重启服务
sudo systemctl restart asset-management

# 查看日志
sudo journalctl -u asset-management -f
```

## 七、常见问题排查

### 7.1 端口被占用
```bash
# 查看 8080 端口占用
netstat -tlnp | grep 8080

# 修改端口：编辑 application.yml
server:
  port: 8081
```

### 7.2 数据库连接失败
- 检查 MySQL 是否运行：`systemctl status mysql`
- 检查防火墙：`sudo ufw allow 3306`
- 检查用户名密码是否正确
- 检查数据库是否存在：`SHOW DATABASES LIKE 'asset_db';`

### 7.3 内存不足
```bash
# 调整 JVM 参数
java -Xms512m -Xmx1g -jar target/asset-management-1.0.0.jar
```

### 7.4 日志级别调整
编辑 `application.yml`：
```yaml
logging:
  level:
    com.asset: DEBUG  # 改为 INFO 减少日志
```

## 八、备份与恢复

### 8.1 数据库备份
```bash
# 备份整个数据库
mysqldump -u root -p asset_db > asset_db_backup_$(date +%Y%m%d).sql

# 定时备份（crontab）
0 2 * * * mysqldump -u root -p'yourpassword' asset_db > /backup/asset_db_$(date +\%Y\%m\%d).sql
```

### 8.2 数据库恢复
```bash
mysql -u root -p asset_db < asset_db_backup_20240101.sql
```
