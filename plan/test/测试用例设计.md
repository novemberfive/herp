# 资产管理系统测试用例设计

## 1. 测试策略与范围

### 1.1 测试目标
- 验证系统功能符合《界面规格说明书》和《业务流程图》定义
- 确保主流程畅通，异常流程有合理处理
- 覆盖边界值场景，防止数据溢出和逻辑错误
- 作为需求完善度的"试金石"，发现设计缺口

### 1.2 测试范围
| 模块 | 测试重点 | 优先级 |
|------|---------|--------|
| 认证模块 | 登录、权限控制、Token 刷新 | P0 |
| 基础信息管理 | 分类/位置/主数据的 CRUD、导入导出 | P0 |
| 资产取得 | 采购申请→入库全流程 | P0 |
| 资产档案 | 卡片管理、状态变更 | P0 |
| 资产管理 | 领用、移交、维修、处置流程 | P0 |
| 资产盘点 | 任务创建→扫码盘点→差异处理 | P0 |
| 报表统计 | 数据准确性、聚合逻辑 | P1 |
| 预警系统 | 规则触发、通知推送 | P1 |

### 1.3 测试类型
- **功能测试**：验证业务逻辑正确性
- **边界测试**：验证极限值处理
- **异常测试**：验证错误处理和容错能力
- **接口测试**：验证 API 契约遵守情况
- **UI 交互测试**：验证页面元素和交互行为
