# 医院资产管理系统 - 非功能需求说明

## 1. 性能需求

### 1.1 响应时间
| 业务场景 | 响应时间要求 | 说明 |
|---------|------------|------|
| 页面加载 | ≤ 2 秒 | 常规页面首次加载时间 |
| 查询操作 | ≤ 3 秒 | 包含复杂条件的列表查询 |
| 数据导入 | ≤ 5 秒/千条 | 批量数据导入处理时间 |
| 数据导出 | ≤ 10 秒/万条 | 大数据量导出处理时间 |
| 扫码盘点 | ≤ 1 秒 | 单次扫码识别响应时间 |
| 实时定位 | ≤ 2 秒 | 资产位置更新延迟 |
| 报表生成 | ≤ 5 秒 | 常规统计报表生成时间 |
| 预警检测 | ≤ 30 秒 | 全量资产预警条件扫描周期 |

### 1.2 并发能力
- **并发用户数**: 支持至少 500 个并发用户同时在线操作
- **峰值处理能力**: 支持每秒 100 次以上的事务处理请求
- **批量操作**: 支持单次 10,000 条数据的批量导入/导出
- **物联网设备**: 支持至少 5,000 个 IoT 设备同时上报位置数据

### 1.3 数据处理能力
- **资产规模**: 支持管理不少于 100 万条资产记录
- **历史数据**: 支持 10 年以上历史数据的存储和查询
- **日志记录**: 支持不少于 3 年的操作日志存储
- **附件存储**: 支持单个文件最大 100MB 的附件上传

---

## 2. 可靠性需求

### 2.1 系统可用性
- **可用性指标**: 系统年可用率 ≥ 99.9%
- **计划维护**: 每月计划维护时间不超过 4 小时，需在非工作时间进行
- **故障恢复**: 一般故障恢复时间 ≤ 30 分钟，严重故障 ≤ 2 小时
- **服务连续性**: 支持 7×24 小时不间断运行

### 2.2 数据可靠性
- **数据完整性**: 确保数据在传输和存储过程中的完整性，数据丢失率 < 0.001%
- **事务一致性**: 所有数据库操作必须满足 ACID 特性
- **数据校验**: 关键业务数据必须进行多重校验，防止错误数据入库
- **异常处理**: 系统异常时必须保证数据回滚，不产生脏数据

### 2.3 容灾备份
- **备份策略**: 
  - 实时备份：关键业务数据实时同步
  - 增量备份：每小时进行一次增量备份
  - 全量备份：每天凌晨进行一次全量备份
- **备份保留**: 备份数据保留不少于 90 天
- **灾难恢复**: RTO(恢复时间目标) ≤ 4 小时，RPO(恢复点目标) ≤ 1 小时
- **异地容灾**: 支持异地数据备份和灾难恢复能力

## 3. 安全性需求

### 3.1 身份认证
- **多因素认证**: 支持密码 + 短信/邮箱验证码的多因素认证
- **单点登录**: 支持与医院统一身份认证系统集成
- **会话管理**: 用户会话超时自动登出（默认 30 分钟无操作）
- **登录限制**: 支持 IP 限制、登录失败次数限制（5 次失败锁定 30 分钟）

### 3.2 权限控制
- **角色管理**: 支持基于角色的访问控制（RBAC）
- **数据权限**: 支持按部门、岗位的数据访问权限隔离
- **功能权限**: 细粒度到按钮级别的功能权限控制
- **权限审计**: 所有权限变更必须记录审计日志

### 3.3 数据安全
- **传输加密**: 所有数据传输采用 HTTPS/TLS 1.2+ 加密
- **存储加密**: 敏感数据（如个人信息）必须加密存储
- **脱敏展示**: 敏感信息在前端展示时进行脱敏处理
- **防泄漏**: 导出数据需审批，并添加水印和追踪标识

### 3.4 安全审计
- **操作日志**: 记录所有用户的增删改查操作，包括操作人、时间、IP、内容
- **登录日志**: 记录所有登录行为，包括成功和失败的登录尝试
- **审计报表**: 提供安全审计报表，支持异常行为分析
- **日志保护**: 审计日志不可篡改，保存期限不少于 3 年

### 3.5 网络安全
- **防火墙**: 部署 Web 应用防火墙（WAF）
- **入侵检测**: 部署入侵检测和防御系统（IDS/IPS）
- **漏洞扫描**: 定期进行安全漏洞扫描和渗透测试
- **DDoS 防护**: 具备分布式拒绝服务攻击防护能力

## 4. 可扩展性需求

### 4.1 架构扩展
- **水平扩展**: 支持通过增加服务器节点实现横向扩展
- **微服务架构**: 核心功能模块支持独立部署和扩展
- **负载均衡**: 支持负载均衡，动态分配请求到不同节点
- **容器化**: 支持 Docker 容器化部署，便于弹性伸缩

### 4.2 功能扩展
- **插件机制**: 支持通过插件方式扩展新功能
- **配置化**: 业务流程、表单字段支持配置化，减少代码修改
- **工作流引擎**: 内置工作流引擎，支持自定义审批流程
- **规则引擎**: 支持业务规则的灵活配置和调整

### 4.3 数据扩展
- **分库分表**: 支持数据库分库分表，应对大数据量场景
- **读写分离**: 支持数据库读写分离，提升查询性能
- **缓存机制**: 支持多级缓存（Redis、本地缓存）
- **数据归档**: 支持历史数据自动归档，减轻主库压力

### 4.4 接口扩展
- **API 网关**: 提供统一的 API 网关，支持接口版本管理
- **开放平台**: 提供开放 API，支持第三方系统集成
- **Webhook**: 支持 Webhook 机制，实现事件驱动的数据同步
- **SDK 支持**: 提供主流编程语言的 SDK

## 10. 验收标准

- 通过压力测试工具验证并发用户数和响应时间指标
- 大数据量场景下进行性能测试，确保系统稳定运行
- 持续运行 72 小时稳定性测试，无内存泄漏和性能衰减

### 10.2 安全验收
- 通过第三方安全机构的渗透测试和安全评估
- 完成漏洞扫描，高危漏洞清零
- 通过等级保护测评

### 10.3 功能验收
- 所有功能模块通过单元测试、集成测试和用户验收测试
- 关键业务流程端到端测试通过率 100%
- 用户验收测试问题修复率 100%

### 10.4 文档验收
- 提供完整的需求文档、设计文档、测试文档
- 提供用户手册、运维手册、API 文档
